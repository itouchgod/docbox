# 文档自动保存系统使用指南

## 1. 系统概述

这是一个用于快速保存和管理各类文档的系统，支持以下四个分类：
- 工作文档 (work)
- 学习笔记 (study)
- 生活记录 (life)
- 兴趣爱好 (hobby)

## 2. 快速开始

### 2.1 保存命令
```bash
savework   # 保存工作相关内容
savestudy  # 保存学习相关内容（默认）
savelife   # 保存生活相关内容
savehobby  # 保存兴趣相关内容
```

### 2.2 推送命令
```bash
pushall    # 一次性推送所有更改到 GitHub
```

## 3. 使用流程

1. **复制内容**
   - 选中要保存的内容
   - 使用 Cmd+C 复制到剪贴板

2. **执行保存**
   - 根据内容类型，使用对应的保存命令
   - 系统会自动：
     - 创建带时间戳的 markdown 文件
     - 添加文件到 Git
     - 进行本地提交

3. **推送更改**
   - 使用 `pushall` 命令推送所有更改
   - 命令会自动：
     - 添加所有更改 (git add .)
     - 创建提交 (git commit)
     - 推送到远程 (git push)

## 4. 目录结构

```
topsir/
├── work/docs/    # 工作文档
├── study/docs/   # 学习笔记（默认）
├── life/docs/    # 生活记录
└── hobby/docs/   # 兴趣爱好
```

## 5. 技术细节

### 5.1 核心文件
- `save_chat.py`: 文档保存和 Git 操作的核心逻辑
- `save.sh`: 命令行接口，支持分类参数

### 5.2 保存格式
每个保存的文件都包含：
- 时间戳文件名 (例如: chat_20241228_120902.md)
- YAML 格式的元数据头部
- 实际内容

## 6. 常见问题

### 6.1 命令不生效
```bash
source ~/.zshrc  # 重新加载配置
```

### 6.2 Git 推送失败
1. 检查 Git 状态：
```bash
git status
```

2. 检查 GitHub 连接：
```bash
ssh -T git@github.com
```

### 6.3 文件位置
所有文件按类别保存在对应的 docs 目录下，可以通过以下命令查看：
```bash
ls -lt work/docs/    # 查看工作文档
ls -lt study/docs/   # 查看学习笔记
ls -lt life/docs/    # 查看生活记录
ls -lt hobby/docs/   # 查看兴趣爱好
```

## 7. 注意事项

1. 确保 Git 和 SSH 已正确配置
2. 确保 Python 3 已安装
3. 使用前先 `pushall` 同步最新内容
4. 定期检查 GitHub 连接状态
5. 重要文档建议额外备份
